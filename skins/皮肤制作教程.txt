萌创匠盒输入法 - 皮肤制作教程（本仓库开源，大家一起维护）
============================================================

一、原则：一个皮肤一个文件夹，文件夹名即皮肤名
------------------------------------------------
• 在 skins 目录下，每个子文件夹 = 一个皮肤。
• 文件夹名就是该皮肤的名字（APK 里显示的名字，也会被远程拉取时当作皮肤 ID）。
• 应用内皮肤列表为「两列一排」上下排列（每行 2 个皮肤）。

二、每个皮肤文件夹里的两个总分类
--------------------------------
打开某个皮肤文件夹后，里面只放两个子文件夹：

  1) 键盘图片文件夹（放该皮肤的所有按键图、背景图）
  2) 介绍文件夹（放该皮肤的视频介绍、图片介绍、文字介绍等，点开皮肤后可查看/下载）

三、键盘图片文件夹内部结构（四个子类型）
----------------------------------------
在「键盘图片」这个文件夹里，再建以下子文件夹，并放入对应图片：

  • background（背景图）
    放一张键盘整体背景图，文件名建议：background.png
    推荐尺寸：720×320 像素（或宽高比约 2.25:1～3:1），会拉伸铺满四种键盘及底部横条。

  • k26（26 键）
    只要打开的是 26 键键盘，界面上出现的每一个键都要有图：除了 26 个字母键以外，删除键、换行键、中/英、逗号、句号、空格、123、Shift 等全部算在内。文件名 = 按键 ID + .png，见下面「26 键按键命名表」。

  • k9（九宫格）
    换成九宫格键盘后，除了中间 9 个数字格以外，符号、123、空格、中/英、退格、换行等所有键都要有图。文件名见「九宫格按键命名表」。

  • handwrite（手写）
    手写键盘上只要有一个键，就要有对应的按键图。文件名见「手写按键命名表」。

  • stroke（笔画/偏旁/生僻字）
    笔画（偏旁）输入法上每一个键都要有图。文件名见「笔画按键命名表」。

四、按键图片命名表（文件名 = 下表 ID + .png，例如 k26_q.png）
------------------------------------------------------------
以下为各键盘上的「全部」按键（26 键 = 字母 + 删除/换行/中英/标点等；九宫格 = 9 格 + 符号/123/空格等；手写/笔画 = 所有可见键）。

【26 键】k26/（26 个字母 + 删除、换行、中/英、逗号、句号、空格、123、Shift 等）
  k26_q, k26_w, k26_e, k26_r, k26_t, k26_y, k26_u, k26_i, k26_o, k26_p,

五、介绍文件夹（视频、图片、文字）
---------------------------------
在「介绍」文件夹里可随意放置：
  • 视频：用于皮肤介绍/使用教程（如 .mp4）
  • 图片：截图、预览图等
  • 文字：如 readme.txt、使用说明.txt 等
运行「本皮肤专用脚本」后，会把这些内容整理成可被 APK 拉取的介绍数据，用户点开该皮肤后可查看、下载或引用。

六、两个 Python 脚本的使用顺序
------------------------------

(1) 本皮肤专用脚本（build_skin.py）
  • 把 skins/build_skin.py 复制到「当前这个皮肤文件夹」里（和「键盘图片」「介绍」两个文件夹同级）。
  • 放好键盘图片、介绍内容后，在该皮肤文件夹内运行：python build_skin.py。
  • 脚本会检测本文件夹内的「键盘图片」和「介绍」内容，生成：
    - 本皮肤的键盘使用教程数据（intro.json）
    - 本皮肤可被 APK 拉取并正常安装使用的 skin.json。

(2) 仓库根目录共用脚本（build_skins.py）
  • 在本仓库根目录下有一个共用的 build_skins.py。
  • 做完一个或多个新皮肤并都运行过各自的 build_skin.py 后，回到仓库根目录，运行：python build_skins.py。
  • 它会扫描 skins/ 下所有皮肤文件夹（以文件夹名为皮肤名），生成 skins/list.json 与默认白色壁纸；APK 拉取后应用内会以「两列一排」方式展示，点开可查看该皮肤的介绍（图片、视频、文字等）并下载/应用。

七、推荐目录结构示例
--------------------
skins/
  纯白/                          ← 皮肤名 = 文件夹名（white 已内置）
    intro/                       ← 介绍：视频、图片、文字
      readme.txt
      preview.mp4
    assets/                      ← 键盘图片（或你命名的「键盘图片」文件夹）
      background/
        background.png
      k26/
        k26_q.png, k26_w.png, ...
      k9/
        k9_1.png, k9_2.png, ...
      handwrite/
        handwrite_clear.png, ...
      stroke/
        stroke_1.png, stroke_2.png, ...
    build_skin.py                 ← 从 skins/ 复制的脚本，放这里后运行

仓库根目录/
  build_skins.py                 ← 共用脚本，运行后汇总所有皮肤并生成 list.json

八、应用内使用方式（不强制拉取）
--------------------------------
• 皮肤在列表里只展示，不会一进来就强行下载。
• 只有用户点开该皮肤、查看介绍、点击「下载」后，才会把该皮肤下到本机。
• 用户再点击「应用」，才会把已下载的皮肤应用到当前键盘上。

九、总结
--------
• 一个皮肤一个文件夹，文件夹名 = 皮肤名。
• 皮肤文件夹内：一个放「键盘图片」（再分 background、k26、k9、handwrite、stroke，且各键盘为全部按键），一个放「介绍」。
• 按键图命名按上表（如 k26_q.png、k9_1.png）。
• 先在该皮肤文件夹里运行 build_skin.py，生成该皮肤的 skin.json 与 intro.json；再在仓库根目录运行 build_skins.py，让 list.json 包含新皮肤，提交 PR 后大家即可拉取到新皮肤。
